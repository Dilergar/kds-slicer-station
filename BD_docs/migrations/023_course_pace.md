# Миграция 023: «Шаг курса» для умной очереди v2 («Темп курсов»)

## Дата выполнения
2026-07-07

## Файл
`server/migrations/023_course_pace.sql`

## Что делает
Добавляет колонку `course_pace_seconds INT NOT NULL DEFAULT 120` в `slicer_settings`.

## Зачем
Умная очередь (`buildSmartQueue` в `smartQueue.ts`) переработана с волновой FIFO-симуляции на модель **«Темп курсов»** (дизайн владельца, 2026-07-07). Модель решает две проблемы старой очереди:
- большой стол «съедал» всю очередь своими курсами, маленькие столы (1-2 блюда) ждали слишком долго;
- при попытке «маленькие столы вперёд» большой стол голодал бесконечно при потоке новых заказов.

## Как работает модель «Темп курсов»
- Каждый гость (стол или чек доставки, `waveKey`) движется по своим курсам независимо. Курсы = уникальные категории его позиций, отсортированные по `sort_index` (0, 1, 2…).
- Виртуальное время позиции: `vt = max(время пробития, время первого заказа гостя + номер_курса × course_pace_seconds)`.
- Очередь сортируется по `vt` (тай-брейки: категория → время пробития). Одинаковые блюда вливаются в карточку выше, если это не даёт гостю курс N раньше курса N−1.
- ULTRA — отдельно сверху (FIFO), как во всех режимах.

`course_pace_seconds` = «сколько в среднем гость ест одно блюдо»:
- **больше** (напр. 300) → обслуживание «тактами», короткие заказы выходят раньше, слияние плотнее;
- **меньше** (напр. 30) → ближе к строгому FIFO по времени заказа.

Защита от голодания: `vt` курса N фиксирован (заказ + N×шаг), столы, пришедшие позже этого момента, встают ПОЗЖЕ — второй курс старожила не отодвигается бесконечно потоком новичков.

## Изменения в коде
- `smartQueue.ts` — `buildSmartQueue` полностью переписан на модель «Темп курсов» (4-й параметр теперь `coursePaceMs`, не `courseWindowMs`).
- `server/src/routes/settings.ts` — GET/PUT маппинг `course_pace_seconds` ↔ `coursePaceSeconds`, валидация 10..3600.
- `services` / `types.ts` — поле `SystemSettings.coursePaceSeconds`.
- `components/SlicerStation.tsx` — передаёт `coursePaceMs` в `buildSmartQueue`.
- `components/admin/SystemSettingsTab.tsx` — UI-поле «⏱️ Шаг курса (секунды)» под тумблером «Волновая Агрегация (Умная)» с полным описанием.
- `App.tsx` — дефолт `coursePaceSeconds: 120` в инертном начальном стейте.

## Примечание про `course_window_seconds`
Колонка `course_window_seconds` (миграция 001) больше НЕ используется умной очередью — та перешла на `course_pace_seconds`. `course_window_seconds` осталась для СТАНДАРТНОЙ сортировки (оба режима агрегации выключены) как размер FIFO-bucket'а правила COURSE_FIFO.

## Откат
```sql
ALTER TABLE slicer_settings DROP COLUMN IF EXISTS course_pace_seconds;
```
Фронт при отсутствии поля использует фолбэк 120 сек (`settings?.coursePaceSeconds || 120`).
