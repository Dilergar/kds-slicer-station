# Миграция 019 — Авто-парковка десертов по модификатору времени

**Дата:** 2026-04-23
**Файл:** `server/migrations/019_dessert_modifier_trigger.sql`

## Цель

Уточнить правило авто-парковки десертов (миграция 017): теперь паркуется **не любая** позиция дессертной категории, а только та, у которой официант в кассе выставил **модификатор связанный со временем**. Без такого модификатора — десерт идёт в очередь нарезчика сразу, как и другие блюда.

Это отражает реальность: гости иногда просят «несите сразу», иногда «попозже». 017-я паркинговала всех подряд.

## Изменение схемы

### `slicer_settings` — добавлена колонка

| Колонка | Тип | Default | Назначение |
|---|---|---|---|
| `dessert_trigger_modifier_patterns` | TEXT[] NOT NULL | `{Готовить%, Ждать%}` | Паттерны LIKE (case-insensitive) для имён модификаторов из `ctlg20_modifiers`. При наличии хотя бы одного такого модификатора у дессертной позиции — авто-парковка срабатывает. |

## Источники данных в чужой БД

- `ctlg20_modifiers` (справочник модификаторов) — имя модификатора, например `"Готовить к 18.00"`, `"Готовить позже"`, `"Ждать разъяснений"`.
- `docm2tabl2_dishmodifiers` — связка позиции и модификатора: `docm2tabl2_itemrow` = `docm2tabl1_items.suuid`, `docm2tabl2_ctlg20_uuid__modifier` = `ctlg20_modifiers.suuid`. Одна позиция может иметь несколько модификаторов.

Чужие таблицы только читаются.

## Логика `unpark_at`

В `GET /api/orders` (блок авто-парковки):

1. `WITH trigger_mods` — для каждой позиции собираем `MAX(minutes_of_day)` по имени модификатора вида `"…к HH.MM"` / `"…к HH:MM"` (regex `к\s*(\d{1,2})[.:](\d{2})`). Если несколько таких — берём самое позднее время (консервативно).
2. `INSERT INTO slicer_order_state` добавляется `INNER JOIN trigger_mods` — позиции без триггер-модификатора просто не попадают.
3. `unpark_at` вычисляется так:
   - `minutes_of_day NOT NULL` → `CURRENT_DATE + minutes_of_day min` (до указанного времени сегодняшней даты). Если момент уже в прошлом — auto-unpark блок в том же GET сразу переведёт в `ACTIVE`.
   - `minutes_of_day IS NULL` (например «Готовить позже», «Ждать разъяснений») → `ordertime + dessert_auto_park_minutes` (default 40 мин).

## Rollback

```sql
ALTER TABLE slicer_settings DROP COLUMN IF EXISTS dessert_trigger_modifier_patterns;
```

После отката логика возвращается к «любой десерт паркуется» (миграция 017) — уточните, нужно ли это или и 017 откатываете.
