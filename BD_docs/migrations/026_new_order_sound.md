# Миграция 026: звуковой сигнал при поступлении нового заказа

## Дата выполнения
2026-07-12

## Файл
`server/migrations/026_new_order_sound.sql`

## Что делает
Одна команда:
```sql
ALTER TABLE slicer_settings
  ADD COLUMN IF NOT EXISTS enable_new_order_sound BOOLEAN NOT NULL DEFAULT TRUE;
```
Добавляет в singleton-таблицу настроек глобальный тумблер звука нового
заказа. `IF NOT EXISTS` — миграция идемпотентна, повторный запуск безопасен.

**Чужие таблицы не затрагиваются.**

## Зачем
До этой миграции единственным звуком модуля был сигнал готовности
разморозки (миграция 016). Владелец попросил отдельный сигнал на появление
нового заказа: в шумной кухне нарезчик не смотрит в планшет постоянно и
замечал новые позиции с опозданием.

## Как работает (логика на фронте, БД хранит только тумблер)
- `SlicerStation` держит множество всех id заказов, виденных за жизнь
  компонента (`knownOrderIdsRef`). Поллинг `GET /api/orders` каждые 4 сек;
  появление id, которого нет в множестве, → двойной beep
  (`playNewOrderBeep()` в `utils.ts`, Web Audio, без внешних зависимостей).
- Первый снапшот после загрузки/F5 запоминается **молча** — иначе каждый
  вход на доску «звенел» бы всеми текущими заказами.
- Множество не чистится при исчезновении заказа: «Готово → Вернуть»
  (restore) возвращает тот же id и повторно не звучит.
- Один beep на тик поллинга, сколько бы позиций ни пришло разом (чек на
  5 блюд — один сигнал, а не пять).
- Паттерн звука (два одинаковых высоких тона) отличается от 3-тонального
  нарастающего сигнала разморозки — события различимы на слух.

## UI
Админка → Общие Настройки → «Звук при новом заказе» (ВКЛ/ВЫКЛ), рядом с
тумблером звука разморозки. По умолчанию ВКЛ.

## Связанные изменения в коде (той же датой)
- `utils.ts` — новая функция `playNewOrderBeep()`.
- `components/SlicerStation.tsx` — эффект `knownOrderIdsRef` + проп
  `ordersLoading` (флаг первой загрузки из `useOrders`).
- `App.tsx` — прокидывает `loading` из `useOrders` в `SlicerStation`.
- `server/src/routes/settings.ts` — маппинг `enable_new_order_sound` ↔
  `enableNewOrderSound` в GET/PUT.
- `types.ts` — поле `enableNewOrderSound?: boolean` в `SystemSettings`.
- `components/admin/SystemSettingsTab.tsx` — тумблер.

## Откат
```sql
ALTER TABLE slicer_settings DROP COLUMN IF EXISTS enable_new_order_sound;
```
