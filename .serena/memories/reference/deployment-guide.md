# Инструкция по развёртыванию модуля (Инструкция.md)

## Файл
`Инструкция.md` в корне проекта — пошаговый гайд для IT-команды заказчика,
как развернуть модуль нарезчика на существующую продакшн-БД `arclient`.

## Цель
Готовый deliverable, который идёт в zip вместе с кодом. Программист заказчика
открывает Инструкцию и проходит её сверху вниз без чтения остального кода —
модуль должен заработать.

## Структура разделов
1. **Требования** — версии PG/Node, права пользователя БД
2. **Настройка UUID цеха кухни** — самый важный раздел, идёт первым после требований (молчаливый баг)
3. **Миграции модуля нарезчика** — 4 миграции 001→004 с командами psql
4. **Конфигурация backend** — шаблон `.env`, `npm install`, тест подключения к БД
5. **Запуск** — dev + production сборка
6. **Проверка работоспособности** — чек-лист по backend API и frontend вкладкам
7. **Начальная настройка модуля через UI** — 7 шагов для первого дня работы нарезчика
8. **Возможные проблемы** — FAQ собранный по реальной симуляции
9. **История симуляции развёртывания** — отчёт какие баги нашли

## Контекст создания
Создан в результате симуляции передачи модуля программистам заказчика.
Пользователь (Sanzhar) получил актуальный прод-дамп `dump-arclient-202604141228`
(PG 9.6, 203 MB), стёр локальную `arclient`, восстановил из дампа, прошёл всё
с нуля как программист заказчика, и фиксировал каждый работающий шаг в Инструкцию.

## Связанные файлы
- `server/.env.example` — шаблон конфига, создан в рамках этой же работы
- `server/package.json` → `migrate` script — обновлён чтобы прогонять все 4 миграции (была 001+002)
- `CLAUDE.md` — файловое дерево обновлено под 4 миграции

## При обновлении модуля
Если меняются:
- Миграции → обновить раздел «Миграции модуля нарезчика»
- `.env` переменные → обновить раздел «Конфигурация backend» + `server/.env.example`
- `KITCHEN_STORAGE_UUIDS` логика → обновить раздел «Настройка UUID цеха кухни»
- API endpoints или вкладки UI → обновить раздел «Проверка работоспособности»
